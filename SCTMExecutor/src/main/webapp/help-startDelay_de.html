Wenn sie mehr als eine Ausführungsdefinition zur Ausführung configuriert haben, können sie hiermit eine kaskadierende Verzögerung der zweiten und aller folgenden Ausführungsdefinitionen in Sekunden spezifizieren.
Das verzögerte Ausführen kann helfen um die Last beim Starten (z.B.: einer virtuallen Maschine) einer Ausführungsdefinition zu minimieren.
Notiz: Wenn eine parallele Ausführung nicht möglich ist wird empfohlen ein weiteres Build Verfahren (Build Trigger) anzulegen.
