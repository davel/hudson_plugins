Enter an existing execution definition id you configured SilkCentral TestManager system. You can enter more than one id by using a comma separated list of ids (e.g. 12345,3389). The execution order of the specified execution definition cannot be affected. Look into the documentation of SilkCentral TestManager for more information about this topics. The specified ids must be in the same project. To start more than one execution definition in different projects, simply add another builder configuration.
